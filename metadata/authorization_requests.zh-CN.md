# 授权申请模板

以下模板用于向上游数据集维护者申请再分发授权。建议通过 GitHub issue、项目维护者邮箱或论文通信作者邮箱发送。

## 中文模板

标题：Request for permission to redistribute processed metadata/samples derived from your smart contract vulnerability dataset

您好，

我是 Daming Xu，正在整理一个面向智能合约漏洞检测研究的公开数据集 `smart-contract-vuln-dataset`。该数据集整合多个公开来源，用于漏洞存在性判断、漏洞类型分类和漏洞行定位任务。

我们的发布策略是：

1. 不再分发贵项目的原始 raw 数据。
2. 保留贵项目名称、URL、引用信息和许可证说明。
3. 优先发布标签、统计、schema、来源指针和重建脚本。
4. 如果获得许可，才发布包含源码片段或处理后源码样本的 processed 数据。

请问贵方是否允许我们在公开 GitHub 仓库中发布从贵项目派生的 processed 样本或元数据？如果允许，是否有指定的引用格式、许可证要求或使用限制？

非常感谢。

Daming Xu

## English Template

Subject: Permission request to redistribute processed metadata/samples derived from your smart contract vulnerability dataset

Dear maintainers,

My name is Daming Xu. I am preparing a public research dataset named `smart-contract-vuln-dataset` for smart contract vulnerability detection, vulnerability type classification, and vulnerable line localization.

Our release policy is:

1. We will not redistribute your raw upstream data.
2. We will preserve your project name, URL, citation, and license notice.
3. We prefer to publish labels, statistics, schema, source pointers, and reconstruction scripts.
4. We will only publish processed samples containing source code if we have explicit permission.

Could you please confirm whether we may redistribute processed samples or metadata derived from your dataset in a public GitHub repository? If permission is granted, please let us know the required citation format, license notice, or any usage restrictions.

Thank you very much.

Daming Xu
